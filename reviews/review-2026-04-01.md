## Befunde

### ConfigService

#### A) Cache wird nie invalidiert 🟡

`ConfigService` cached Config-Werte in `$cache`, aber der Cache wird nie geleert. Da der Service als Singleton registriert ist (Symfony-Standard), bleiben gecachte Werte für die gesamte Request-Dauer bestehen. Das ist bei einem normalen Storefront-Request kein Problem, aber bei CLI-Commands oder Tests mit mehrfachen Config-Änderungen kann es zu veralteten Werten kommen.

**Empfehlung:** Für ein reines Checkout-Plugin ist das akzeptabel. Kein Handlungsbedarf, da Shopware den SystemConfigService selbst cached und die Plugin-Config nur einmal pro Request gelesen wird.

---

### CheckoutSubscriber

#### B) Unreachable Default-Branch im match-Ausdruck 🟢

```php
$step = match (true) {
    $event instanceof CheckoutCartPageLoadedEvent => 1,
    $event instanceof CheckoutRegisterPageLoadedEvent => 2,
    $event instanceof CheckoutConfirmPageLoadedEvent => 3,
    $event instanceof CheckoutFinishPageLoadedEvent => 4,
    default => 1,
};
```

Der `default`-Branch ist unerreichbar, da der Union-Type-Hint alle Fälle abdeckt. Kein funktionales Problem, aber streng genommen toter Code.

**Empfehlung:** Kann bleiben als defensiver Fallback. Minor-Befund, kein Handlungsbedarf.

---

### Templates

#### C) Confirm-Template: `page_checkout_confirm_container` Block-Override mit Layout-Änderung 🟡

```twig
{% block page_checkout_confirm_container %}
    <div class="row">
        <div class="{% if page.extensions.rcCheckoutEnhancer.miniCartEnabled %}col-lg-8{% else %}col-12{% endif %}">
            {{ parent() }}
        </div>
        ...
    </div>
{% endblock %}
```

Dieser Block wickelt den gesamten `parent()`-Inhalt in ein zusätzliches Row/Col-Layout. Wenn ein anderes Plugin denselben Block überschreibt, kommt es zu Konflikten. Das ist ein bekanntes Risiko (bereits in `status.md` dokumentiert).

**Empfehlung:** Dokumentiert und akzeptiert. Falls Konflikte auftreten: eigenen Block innerhalb von `base_main_inner` statt `page_checkout_confirm_container` verwenden.

#### D) Order-Summary: Inline-Style im Mini-Warenkorb 🟢

```twig
<div class="rc-checkout-mini-cart__meta text-muted" style="font-size: 0.8em;">
```

Ein Inline-Style statt SCSS-Klasse. Funktional korrekt, aber inkonsistent mit dem Rest des Codes der ausschließlich SCSS verwendet.

**Empfehlung:** In die SCSS-Datei auslagern als `.rc-checkout-mini-cart__meta { font-size: 0.8em; }`.

**Empfehlung:** `.gitignore-template` als `.gitignore` aktivieren und Shopware-spezifische Einträge ergänzen.

Die `.editorconfig` ist vorhanden aber untracked. Sollte committet werden für konsistente Formatierung im Team.
